1. RabbitMQ-Konfiguration (localhost:15672):
- Login: guest/guest
- Queue: learning_analytics

2. nifi-Konfiguration (localhost:8080):
A) Konfigurationsdatei übernehmen
    von HOST:container/learning-analytics/nifi-conf/flow.xml.gz
    nach CONTAINER:/opt/nifi/nifi-current/conf
  und mounten (passiert schon über docker-compose.yml):
    von HOST:container/learning-analytics/drivers
    nach CONTAINER:/opt/nifi/nifi-current/drivers
B) Manuell importieren mit
    HOST:container/learning-analytics/nifi-conf/Postgres_Flow.xml

Achtung: Das Passwort für die JDBC/Postgres Connection wurde beim Import nicht übernommen. Evtl. klappt es mit dem Import der GZ-Datei, sonst gibht es dafür bestimmt eine Umgebungsvariable oder so.

3. Postgres-Konfiguration:
CONTAINER:
	psql -u postgres
	> CREATE DATABASE nifi;
    > \c nifi;
	> CREATE TABLE flow_files (
		  uuid VARCHAR(255) PRIMARY KEY,
		  filename VARCHAR(255),
		  datetime TIMESTAMP NOT NULL,
		  raw JSON,
		  actor VARCHAR(255),

		  object VARCHAR(255),
		  context VARCHAR(255)

4. Zeppelin-Konfiguration (localhost:8090):
- Interpreter
    von HOST:container/learning-analytics/zeppelin-conf/interpreter.json (relevant ist darin eigentlich nur „nifi”)
    nach CONTAINER:/zeppelin/conf
- Notebooks
    A) Über Github synchronisieren
        - Github Access Key hinterlegen (siehe https://zeppelin.apache.org/docs/0.8.0/setup/operation/configuration.html#create-github-access-token)
        	in HOST:container/learning-analytics/zeppelin-conf/zeppelin-env.sh
        	Setting: ZEPPELIN_NOTEBOOK_GIT_REMOTE_ACCESS_TOKEN
	    - Konfigurationsdatei übernehmen
	    	von HOST:container/learning-analytics/zeppelin-conf/zeppelin-env.sh
	    	nach CONTAINER:/zeppelin/conf
    B) Über REST-API anlegen
    	- Neue Note: https://zeppelin.apache.org/docs/0.7.0/rest-api/rest-notebook.html#create-a-new-note
    	- Neuen Paragraph: https://zeppelin.apache.org/docs/0.7.0/rest-api/rest-notebook.html#create-a-new-paragraph
    C) Konfigurationsdatei übernehmen
    	von HOST:container/learning-analytics/zeppelin-conf/2EZMATNCU
    	nach CONTAINER:/zeppelin/notebook/
